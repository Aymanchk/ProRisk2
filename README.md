# ProRiski — Калькулятор риска (тестовое)

Правая панель калькулятора риска из макета ProRiski: три варианта рынка —
**Рынок США (акции)**, **Криптовалюта**, **Форекс**. Форма с живым расчётом размера
позиции, переключением Long/Short (Buy/Sell), режимами ввода стоп-лосса, выбором
соотношения тейк-профита и всеми состояниями (пусто / заполнено / ошибки / загрузка).

## Стек

- **Next.js 16** (App Router) + **React 19** + **TypeScript** (strict)
- **MUI** + Emotion (SSR-кеш через `@mui/material-nextjs`)
- **MobX** (`mobx-react-lite`)
- **next-intl** — RU/EN, локализованное форматирование чисел, ICU-плюрализация
- **Framer Motion** — анимации режимов и переключения рынка
- **SCSS** (модули + общие токены)
- **Vitest** — юнит-тесты расчётного ядра

## Запуск

```bash
npm install
npm run dev      # http://localhost:3000  (/ → /ru)
```

Прочие команды:

```bash
npm run build      # прод-сборка
npm test           # юнит-тесты расчётов (13 тестов)
npm run typecheck  # tsc --noEmit
npm run lint       # eslint
```

## Что реализовано

- **Три рынка** с разной логикой: акции (целые акции, стоимость, плечо), крипта
  (дробное количество, стоимость, плечо, **цена ликвидации**), форекс (лоты, **залог**,
  **стоимость пипа**, плечо 1:N).
- **Направление** Long/Short, для форекса — Buy/Sell.
- **Режимы стоп-лосса**: «В цифрах» (ввод цены), «в %» (раскрывается доп-поле отступа,
  цена стопа считается автоматически), «ATR» (стоп от ATR таймфрейма; поле read-only).
- **Тейк-профит**: соотношение R:R из списка (1, 1.5, 2, 3, 4) либо ручной ввод цены —
  тогда в селекторе показывается рассчитанное соотношение.
- **Живой расчёт**: риск, профит, количество, стоимость/залог, плечо, цена ликвидации,
  стоимость пипа. Числа форматируются по локали.
- **Состояния**:
  - *empty/default* — незаполненные поля дают прочерки в результатах;
  - *filled* — пересчёт на каждый ввод;
  - *hover/focus* — на полях, кнопках, тоглах;
  - *error* — стоп равен входу / стоп с неверной стороны / не число → красная рамка,
    подпись ошибки, кнопка «Активировать» заблокирована;
  - *loading* — скелетон при смене рынка (имитация загрузки котировок) и спиннер на
    кнопке при отправке;
  - предупреждение о цене ликвидации (крипта).
- **Поповер настроек** (шестерёнка): депозит, риск на сделку (%), плечо — именно они
  задают сумму риска `riskAmount = депозит × риск% / 100`.
- **Копирование ссылки** на расчёт (иконка-цепочка) + снэкбары.
- **Адаптив**: фикс-ширина панели на десктопе, на всю ширину на мобайле (проверено от 360px).
- **i18n**: RU/EN, переключатель языка, локализованные числа и плюрализация («20 акций» /
  «20 shares», «$ 3 508» / «$ 3,508»).

## Архитектура

```
src/
  app/[locale]/        # layout + демо-страница (App Router, локализованный роут)
  i18n/                # routing / navigation / request (next-intl)
  proxy.ts             # locale-роутинг (бывш. middleware, переименован под Next 16)
  providers/           # AppProviders (emotion+theme+store), StoreProvider (MobX)
  theme/               # MUI-тема и SCSS-токены (единый источник цветов)
  store/               # RootStore
  shared/
    lib/               # парсинг/форматирование чисел (+ хук useFormat)
    ui/                # UI-kit: NumberField, SelectField, SegmentedToggle, StatPair, LanguageSwitcher
  features/calculator/
    model/             # types, constants, calculations (+ тесты) — чистая логика
    store/             # CalculatorStore (MobX)
    ui/                # RiskCalculator и подкомпоненты
messages/              # ru.json, en.json
```

Расчётное ядро (`features/calculator/model/calculations.ts`) — чистые функции без React и
MobX, поэтому покрыты юнит-тестами. `CalculatorStore` держит «сырые» строки полей (чтобы не
терять промежуточный ввод) и отдаёт распарсенные числа и результат как MobX-computed.

## Принятые решения и допущения

- **Формулы проверены по числам из макета**:
  - акции: вход 175.40, стоп 170.40, риск $100 → 20 акций, стоимость $3 508, профит $200;
  - форекс: 0.50 лота EUR/USD → залог `50000 × 1.0845 / 100 = $542`, пип `50000 × 0.0001 = $5`;
  - тейк 2:1 → 185.40. См. `calculations.test.ts`.
- **Сумма риска** не вводится напрямую, а считается из депозита и риска на сделку (поповер
  настроек), как в полном калькуляторе ProRiski. Дефолт 10 000 × 1% = $100.
- **Моки данных** (живого фида котировок в тестовом нет):
  - значения **ATR** заданы константами по таймфреймам/рынкам (`constants.ts`);
  - **цена ликвидации** — упрощённая изолированная модель с плоской ставкой
    maintenance-margin (0.5%): `вход × (1 − 1/плечо + mmr)` для лонга. На реальных биржах
    ставка тиражная;
  - изменение цены входа (−0.061% у крипты) — статичное демо-значение.
- **Округление акций** — вниз, чтобы не превышать заданный риск.
- **ATR-режим**: один пресет (таймфрейм 1m, период 10, множитель 1.5), стоп считается
  автоматически, поле read-only. Расширяется до выбора таймфрейма без переделки логики.
- **Три варианта рынка** показаны тремя панелями в ряд (как отдельные фреймы в Figma),
  на мобильном — стопкой. Каждая панель — со своим состоянием (отдельный MobX-стор).
- **Переключатель Long/Short** показывается по режиму, как в макетах: скрыт в режиме «В
  цифрах», появляется в «в %» и ATR.
- **Размеры, отступы, цвета и типографика** выверены по Figma (Dev Mode): ширина панели
  312, паддинг 12, бордеры #f0f0f0, зелёный #329a85, текст #4a4a4a, серый #bdbdbd и т.д.
- **Шрифт**: в макете Lato, но у Lato нет кириллицы в вебе (ни Google Fonts, ни Fontsource),
  а интерфейс русский. Использован Open Sans — ближайший гуманистический аналог с кириллицей.
- **Локали**: RU/EN. Инфраструктура next-intl готова к добавлению локалей; арабский/RTL
  в объём тестового не входил (но `html lang` и провайдеры это поддержат).
